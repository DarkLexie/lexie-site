# Lexie Site — трекер проекта

*Этот файл — источник истины по процессу и статусу. Обновлять после каждого полученного ассета/решения.*

## Общая логика работы (не менять без явной просьбы)
Идём **строго по одному ассету за раз**:
1. Даю промпт для ChatGPT под конкретный ассет.
2. Пользователь генерирует, смотрит результат.
3. Не устраивает → говорит, что поправить → корректирую промпт → повтор.
4. Устраивает → пользователь присылает файл → сохраняю в нужную папку проекта.
5. Только после этого переходим к следующему ассету.

Не выдавать все промпты пачкой. Не переходить к следующему ассету без подтверждения текущего.

## Зафиксированные решения
- **Источник истины по дизайну** — мокапы, присланные пользователем (Dark Art home, Light Art home, Artist page). Держим в голове по переписке, на диск не сохраняем.
- **Стек**: ванильный HTML/CSS/JS, никакого Next.js. Деплой — GitHub.
- **Структура — многостраничная**: `Home` / `Gallery` / `Artist` — отдельные html-страницы. `Contacts` — блок на Artist + футер на всех страницах.
- **Hero-фон** — не статичная картинка, а 3 вертикальных видео-акцента на тему, чередуются случайно (и для Light, и для Dark).
- **Портретная рамка** — золотая (Light) / бордовая (Dark), скруглённый прямоугольник с узорными уголками у Light (сдержаннее у Dark), один и тот же ассет используется в тизере на Home и на странице Artist.
- **Reels** — доп. фильтр-чип в галерее на каждой теме (не отдельная страница/пункт меню). Видео проигрывается в лайтбоксе.
- **Категории фильтра Light** (ревизия 2026-07-18, по `Assets for Light account.png`): `All Works · Vintage · Characters · Creatures · Reels · Black and White`. Magical и Cinematic из исходного ТЗ — убраны, все работы с этими тегами переразмечены (30 шт.).
- **Категории фильтра Dark** (ревизия 2026-07-18): все 6 исходных категорий из ТЗ подтверждены реальным распределением тегов (Horror 44, Gothic 37, Surreal 30, Moody 13, Post-Apocalyptic 12, Cinematic Realism 11) + добавлена **Black and White** (12 работ уже помечены, вероятно есть ещё среди первой партии — не проверено визуально) + Reels. Итог: `All Works · Horror · Gothic · Surreal · Post-Apocalyptic · Cinematic Realism · Moody · Black and White · Reels`. **Подтверждено** присланным `Assets for dark account.png` — список чипов совпал 1-в-1, менять не нужно.
- **Hero-подзаголовок Dark** (точный текст с `Assets for dark account.png`): "Visual stories born between light and shadow. Where emotion shapes the unseen." (отличается от Light-версии в тех.задании).
- **Стиль UI Dark** — референс получен, файл на диске с настоящей прозрачностью (альфа проверена): переключатель Dark Art активен по умолчанию (бордовая заливка, луна) / Light Art неактивен (тёмная пилюля, солнце); белый текст навигации/заголовков с бордовым акцентом; тонкие бордовые линии-разделители между пунктами меню — простой CSS, без отдельного графического ассета. В отличие от Light, здесь нет декоративных искр-вставок — decorative-анимация Dark (падающий пепел) реализуется через canvas/JS, а не картинкой.
- **Стиль UI-текста/кнопок Light** — референс получен (`Assets for Light account.png`, файл на диске, с настоящей прозрачностью — проверено через альфа-канал): тёплый золотой градиент текста с мягким свечением для заголовков/навигации; тёмные пилюли с тонкой золотой обводкой (неактивная) / золотая заливка (активная) для кнопок и фильтр-чипов. Текст/кнопки реализуются как HTML/CSS на вёрстке (не картинкой — важно для SEO/accessibility), искра и линия-разделитель — уже готовые вырезанные PNG-ассеты (см. таблицу ассетов).

## Анимации и взаимодействие (зафиксировано 2026-07-18)

### Переход между темами (theme-switch reveal)
Срабатывает при клике по переключателю Light Art / Dark Art (на Home и на Gallery — переключатель один и тот же компонент на обеих страницах).

- **В Dark**: контент "проявляется" из дыма и пепла — реалистичная кинематичная анимация (не мультяшная), дым рассеивается, открывая тёмную тему под собой.
- **В Light**: контент "проявляется" из тёплого солнечного света и мерцающих золотых частиц — вспышка тёплого света стихает, частицы (переиспользуем `sparkle-light.webp`) разлетаются/тают, открывая светлую тему.
- Длительность ~0.8–1.2 сек, one-shot (не зацикленная), поверх текущего контента (canvas-оверлей на всю секцию/экран).
- `prefers-reduced-motion` → анимация отключается, обычный мгновенный кросс-фейд.
- Технически: canvas 2D, соседствует с уже запланированными `AshParticles`/`FirefliesParticles` (тех.задание §10), но это отдельный "разовый" эффект перехода, не фоновая амбиентная анимация.
- **Триггер подтверждён**: играет на каждом клике переключателя, и на Home, и на Gallery (не только при первом выборе стороны).

### Hover на карточках галереи (уточнение к тех.заданию §9.2)
- **Dark**: при наведении — тёмно-красное свечение (box-shadow, `--burgundy`/`--accent-glow`) + светящаяся красная обводка карточки.
- **Light**: при наведении — переливающаяся золотая обводка + мерцающий блик, проходящий по всей карточке по диагонали (классический CSS shine-sweep: псевдоэлемент с градиентом, translate поперёк карточки при hover).
- Работает вместе с уже решённым zoom 3-5% на изображении (тех.задание §9.2).
- **Мобильная версия**: shine-sweep и переход дым/свет — не завязаны на курсор мыши, отработают и на touch (переход — по тапу на переключатель; shine — можно проиграть один раз при появлении карточки во вьюпорте вместо hover, раз на мобильных нет наведения).

## Лайки и счётчик посещений (обсуждается 2026-07-18, статус планирования)
Из хэндоффа прошлой сессии: технически через Supabase (бесплатный тариф).
- **Лайки**: сердечко на карточке работы, клик = +1, повторный клик = снять; защита от накрутки — локальная память браузера посетителя (localStorage), финальный счёт хранится в Supabase по `slug` работы через защищённые RPC-функции (без приватных ключей на фронтенде).
- **Счётчик посещений**: считает заходы на сайт, виден только владелице (не публично на самом сайте — например, отдельная защищённая страница/пароль, или смотреть напрямую в Supabase).
- ⚠️ Бесплатный Supabase-проект "засыпает" после ~7 дней неактивности — нужно будет вручную "будить".
- **Подтверждено**: делаем после вёрстки (как и решали раньше). Аккаунт Supabase ещё не заведён — заведём, когда дойдём до этого шага.

## Вёрстка — статус на 2026-07-18: первая версия готова и протестирована

Собраны и проверены локально (свой HTTP-сервер + браузерные проверки: рендер, клики, консоль без ошибок):
- `index.html`, `gallery.html`, `artist.html` — все три страницы, контент 1-в-1 с ТЗ/мокапами.
- `css/tokens.css`, `base.css`, `layout.css`, `components.css` — токены под обе темы, адаптив, карточки, лайтбокс, hover-эффекты.

- `js/theme-transition.js` — canvas-переход дым/пепел (Dark) и тёплый свет/искры (Light) на каждый клик переключателя, отключается при `prefers-reduced-motion`.
- `js/particles.js` — амбиентный фон (падающий пепел / светлячки), пауза при скрытой вкладке.
- `js/hero-video.js` — случайный выбор одного из 3 акцент-видео на тему, меняется при смене темы.
- `js/gallery.js` — рендер фильтр-чипов и сетки из `works.json` по теме, лайтбокс (next/prev/Esc/клик вне/видео), мобильный триггер shine-эффекта через IntersectionObserver (вместо hover).
- `js/main.js` — sticky-шапка при скролле, мобильное меню-гамбургер.
- Проверено: смена темы, все фильтры (вкл. Reels), лайтбокс для картинок и видео, мобильное меню при ширине 375px — всё работает, консоль без ошибок.
- Локальный сервер для тестов: `.claude/launch.json` → конфиг `lexie-site` (PowerShell HTTP-сервер на порту 8080, т.к. `fetch()` для `works.json` требует настоящий HTTP, не `file://`).
